# 会话日志 Skill (`/log`)

本 Skill 让你在 OpenCode/ClaudeCode/Codex CLI 等Agent 编码工具中使用 `/log` 命令，把当前会话的**原始用户需求**和对项目做出的**主要变更摘要**，记录到当前工作目录下的 `AGENTS_LOG.md` 文件中。

使用目标：
- 每次用户输入 `/log` 时，生成一条新的会话日志，方便日后回顾本次 Codex 会话做了什么。
- 记录**用户最初是如何提需求的**，而不是一堆工具展开或系统说明。
- 用简洁的中文主题概括这次会话的核心任务。

注意：
- 关于“不要包含展开的 SKILL”，如果某个 SKILL 在用户输入之后自动插入了大段说明或模板，不要当成“用户输入”写进日志。
- 不要记录 /log 本身以及对`AGENTS_LOG.md`的变更。



## AGENTS_LOG.md 的位置与总体结构

日志文件位于**当前工作目录**，路径固定为：`./AGENTS_LOG.md`，追加模式。每次追加一个新的 `##` 段落**，不改写既有历史记录。

文件结构示例：

```md
# AGENT LOG
## <YYYY-MM-DD> <会话主题>

### 用户输入
1. ...
2. ...

### 变更记录
1. ...
2. ...
```

## 会话主题（日志条目标题）

当用户输入 `/log` 时，为本次会话生成一个**简洁的中文主题**，放在二级标题中，如 `## 2026-03-20 Fuzzer测试统计分析报告`

## 用户输入段落（### 用户输入）

在该条日志标题下，首先写入一个 `### 用户输入` 段落，列出本会话中该用户的所有原始 Prompt：

```md
### 用户输入
1. 第一个用户输入
2. 第二个用户输入
...
```

要求：
1. 每一项内容应尽量保持**用户原文**：
   - 可以对极长 Prompt 进行适度压缩（例如去掉明显的重复段落或日志噪声）。
   - 但不要擅自改写用户意图或替换成你的总结。
2. 避免包含 SKILL 展开：如果你能明确识别某一段文本是自动展开的 SKILL 说明，不要写入这里。

## 变更记录段落（### 变更记录）

在 `### 用户输入` 之后，写入 `### 变更记录` 段落，简要总结本次会话对项目做的主要变更：

生成规则：

1. 基于**本次 Codex 会话中你实际执行过的操作**进行总结，包括但不限于：
   - 新增或修改的脚本、源代码、配置文件。
   - 修复的 Bug、改进的逻辑、重构的模块。
   - 添加的测试、文档或自动化步骤。
2. 使用 2–6 条有序列表项，每一条是**一句简洁的中文句子**：
   - 每条聚焦一个具体变更点，不要混杂多个无关事项。
   - 推荐格式：
     - `在 <文件路径> 新增 ... 功能，实现 ...`
     - `修复 <模块> 中 ... 的边界情况，避免 ... 问题`
     - `补充 <脚本/文档>，说明 ... 的使用方法`
3. 如果本次会话并未对项目做出任何实质代码变更，可以写一条说明：
   - 例如：`本次会话仅进行需求澄清与方案设计，未修改代码。`
4. 避免写入过于细碎的操作（如单纯查看文件、运行一次命令），只保留对项目有长期价值的变更。

## 执行流程
如果支持子 Agent，则开启子 Agent 并运行 log SKILL。以实现不污染主 Agent 上下文。
如果不支持，则直接运行。

通过以上约定，本 Skill 能够在用户每次调用 `/log` 时，可靠地把**原始需求、变更摘要和会话主题**记录到当前目录的 `AGENTS_LOG.md` 中。
